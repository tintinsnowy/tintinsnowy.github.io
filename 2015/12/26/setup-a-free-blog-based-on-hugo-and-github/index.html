Setup a free blog based on Hugo and Github

2015-12-26 · 988 words · 5 minute read
2015/12

Hello github!

Github is more than a collaborative tool to make your software development more effective. In fact, it is more like a community where you share and contribute your code.

Since we want to build our own blog, so the whole static web should be store in github. Now here we go. sign up in Github

Okay！ you are now a member of github, enjoy your coding journey!

Bravo！ Hugo

A Powerful and Fast Website Engine is Hugo， which is design for go lovers！ Hugo flexibly works with many formats and is ideal for blogs, docs, portfolios and much more. Hugo’s speed fosters creativity and makes building a website fun again.

It helps you to build your blog, you don’t have to know the details of css , js,MySQL etc. It saves you a lot of trouble, and you can concentrate more on your writing and coding.

why is static web?

Because…. * Quick to develop * Cheap to develop * Cheap to host. Those are what we want for blogs!

Usage of Hugo

  1. Install Hugo two chice

  2. **New blog’s framework ** enter your hugo path

    $ cd hugo
$ ls -la

    now you can start your first website. sherry.io should be replaced by your website name

    $ hugo new site sherry.io

    If you find something wrong with your path, then use the absolute path for example:

    $ home/sherry/code/hugo/hugo new site sherry.io

    Then you can see the tree:

      ▸ archetypes/
  ▸ content/
  ▸ layouts/
  ▸ static/
  config.toml

  3. Fill content in blog your articles should be store in the path of : content/post. adll the articles are in markdown format.

    $ cd sherry.io
$ hugo new about.md
$ hugo new post/firstblog.md

    Then open the firstblog.md , you can see the automatically generated file

    +++
date = "2015-12-26T08:36:54-07:00"
draft = true
title = "firstblog"
+++

Now you can add the content into your file:

    +++
    date = "2015-01-08T08:36:54-07:00"
    draft = true
    title = "first"
    +++
    ####Hello Hugo

     1. aaa
     2. bbb
     3. ccc

  1. Find a joyful theme for your blog There are a few opensource-theme . Every theme has detailed description of usage, but NOT all the configurations are the same. You should see the Homepage of what you’ve chosen. For example,

    $ pwd
>/home/sherry/code/hugo/sherry.io
$ mkdir themes
$ cd themes
$ git clone https://github.com/spf13/hyde.git

    now you can see that the theme has been in your themes/

  2. Set configuration of your theme This step is extremely important, different theme has different ways of setting, so it’s your time to design. But before you do, you should read the homepage of your theme.

Hugo can read your configuration as JSON, YAML or TOML. Hugo supp orts parameters custom configuration too. Refer to the theme that you’ve chosen for details.

…….. you can see how it looks like in localhost:(the hugo-icarus is the name of theme, you need to change it), open “http://localhost:1313/"

 sherry@sherry:~/code/hugo/sherry.io$ sudo /home/sherry/code/hugo/hugo server --theme=hugo-icarus --buildDrafts --watch

If you are satisfied with your blog in localhost, you want to display it on web right? Okay do the following steps!

Create your blog’s repository on Github

Aha, I suppose that you are familiar with git. If not, there is a whale of tutorial of git and github. So shall we start?

  1. create a new repository

    User & Organization Pages live in a special repository dedicated to GitHub Pages files. You will need to name this repository with the account name.

    You must use the username.github.io naming scheme. Content from the master branch will be used to build and publish your GitHub Pages site. A repository like joe/bob.github.io will not build a User Pages site. Don’t ask me why… see here .
    When User Pages are built, they are available at http(s)://<username>.github.io.

For example, my account name is tintinsnowy , so I have to name my repository tintinsnowy.github.io

  1. Build your static website

     $hugo --theme=hugo-icarus --baseUrl="http://tintinsnowy.github.io/"

  2. Upload your website

    $ cd public
$ git init
$ git remote add origin https://github.com/tintinsnowy/tintinsnowy.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master

  3. See your website In your github repository setting, you can see this:

Your site is published at http://tintinsnowy.github.io

Now, it seems like the end of our Project. Yet if you wish your blog’s address more unique, you may buy a custom domain.

Buy domain

A few suggestion: 1. godaddy 2. aliyun 3. …..

Setting

  1. Adding a CNAME file to your repository: Notice:

    • In the file name field, type CNAME (with all caps!).
    • use tintinsnowy.com, not https://tintinsnowy.com. Note that there can only be one domain in the CNAME file. (well you should type in your own CNAME, instead of tintinsnowy)

  2. Setting your account

CNAME @ tintinsnowy.github.io.
A www 192.30.252.153
A www 192.30.252.154

Don’t forget the . after tintinsnowy.github.io If you want to know why should be setting like this, see the references.

  1. waiting..

  2. checkout

     tintinsnowy.com
