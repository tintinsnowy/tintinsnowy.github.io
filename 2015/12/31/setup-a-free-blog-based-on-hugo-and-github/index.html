Setup a free blog based on Hugo and Github

2015-12-31 · 628 · 3
2015/12

Hello GitHub!

GitHub is more than a collaborative tool to make your software development more effective. In fact, it is more like a community where you share and contribute your code.

Since we want to build our own blog, so the whole static websites should be stored in Github. Now here we go. sign up in Github

Okay you are now a member of github, enjoy your coding journey!

Bravo Hugo

A powerful and fast Website engine is Hugo， which is designed for go lovers！ Hugo flexibly works with many formats and is ideal for blogs, docs, portfolios and much more. Hugo’s speed fosters creativity and makes building a website fun again.

It helps you to build your blog, you don’t have to know the details of css , js,MySQL etc. It saves you a lot of trouble, and you can concentrate more on your writing and coding.

why is static web?

Because….

  • Quick to develop
  • Cheap to develop
  • Cheap to host.
  • Those are what we want for blogs!

Usage of Hugo

1. Install Hugo

    [Download](https://gohugo.io/)
    tar -zxvf hugo*
    ln -s /home/sherry/hugo_0.17_linux_amd64 hugo
    hugo version

###2. New blog’s framework choose a depository to store your webpages

    $ hugo new site tintinsnowy.github.io(example-site)

Then you can see the tree:

  ▸ archetypes/
  ▸ content/
  ▸ layouts/
  ▸ static/
  config.toml

3. template of blog

here [https://gohugo.io/showcase/](https://gohugo.io/showcase/)

you can find the templates of your blog. Of course you may code yourself

    $ mkdir themes
    $ cd themes
    $ git clone https://github.com/coderzh/hugo-rapid-theme.git

4. content of blog

    $ cd tintinsnowy.github.io
    $ hugo new about.md
    $ hugo new post/firstblog.md

before you edit you should modify the content of config.toml according to the template you use

5. Debug

here you can see the local website of your blog

6.Publish and create your blog’s repository on Github

Aha, I suppose that you are familiar with git. If not, there is a whale of tutorial of git and github. So shall we start?

create a new repository

User & Organization Pages live in a special repository dedicated to GitHub Pages files. You will need to name this repository with the account name.

You must use the username.github.io naming scheme. Content from the master branch will be used to build and publish your GitHub Pages site. A repository like joe/bob.github.io will not build a User Pages site. Don’t ask me why… see here . When User Pages are built, they are available at http(s)://<username>.github.io.

For example, my account name is tintinsnowy , so I have to name my repository tintinsnowy.github.io

Build your static website

    $hugo --theme=hugo-icarus --baseUrl="http://tintinsnowy.github.io/"
    $ cd public
    $ git init
    $ git remote add origin https://github.com/tintinsnowy/tintinsnowy.github.io.git
    $ git add -A
    $ git commit -m "first commit"
    $ git push -u origin master
    See your website

In your github repository setting, you can see this:http://tintinsnowy.github.io

8. Setting

  1. Adding a CNAME file to your repository: Notice:

    • In the file name field, type CNAME (with all caps!).
    • use tintinsnowy.com, not https://tintinsnowy.com. Note that there can only be one domain in the CNAME file. (well you should type in your own CNAME, instead of tintinsnowy)

  2. Setting your account

CNAME @ tintinsnowy.github.io.
A www 192.30.252.153
A www 192.30.252.154

Don’t forget the. after tintinsnowy.github.io If you want to know why should be setting like this, see the references.
